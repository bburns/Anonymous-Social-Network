ASN1

Specification

ASN is an attempt at an anonymous social network that tracks a student's academic interests and displays the academic interests of others.

There are three types of users:

  1. administrators
  2. students
  3. browsers

Administrators can do the following:

  • run unit tests
  • import/export the data
  • remove a user and its data
  • reset the system

Students can do the following:

  • create an account (ID and password randomly generated by the system)
  • log in with randomly generated ID and password
  • change password
  • add/edit/remove classes (unique, course number, course name, semester, instructor, rating, grade)
  • add/edit/remove books (isbn, title, author, rating)
  • add/edit/remove papers (journal or conference, title, author, rating)
  • add/edit/remove internships (company, location, semester, rating)
  • add/edit/remove places to study (location, semester, rating)
  • add/edit/remove places to live (location, semester, rating)
  • add/edit/remove places to eat (location, semester, rating)
  • add/edit/remove places to have fun (location, semester, rating)
  • add/edit/remove games (os, title, rating)

Browsers can see lists of the following and for each, the number of ratings and the average rating:

  • courses (average grade)
  • books
  • papers
  • internships
  • places to study
  • places to live
  • places to eat
  • places to have fun
  • games

In this phase of the project your group needs to do the following:

  • design an XML schema and instance to represent the data
  • create a UML diagram of the data
  • implement a set of Google App Engine models that can then store and retrieve that data in the Google App Engine datastore
  • implement an import/export facility between XML and the Google App Engine datastore
  • have each member of the group create his or her data
  • collaborate with the other groups in the class and arrive at a single XML schema and instance for the whole class
  • import and export that group XML schema and instance on the Google server
  • import and export that unified XML schema and instance, locally

Groups

Pick a member of the group to be project leader. A group member can only be project leader once.

Once the TA has graded the project, that will be the project grade.

The project leader will always receive the project grade.

The project leader can add or subtract up to 10 points to or from any member of the group in exchange for adding or subtracting the same amount of points from the other member(s) of the group.

For example, the leader could add 10 points to person A, and subtract 5 points from person B and person C.

The leader will report these adjustments in the Google Form.
